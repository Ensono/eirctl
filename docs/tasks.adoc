= Tasks
:toc:
:toclevels: 2

Tasks are the fundamental building blocks of eirctl. Each task defines a set of commands to execute.

== Basic Task

[source,yaml]
----
tasks:
  hello:
    command: echo "Hello, World!"
    
  build:
    description: "Build the application"
    command:
      - go build -o bin/app ./cmd/main.go
      - echo "Build complete"
----

== Task Properties

[cols="2,1,3",options="header"]
|===
| Property | Type | Description

| `command`
| string or array
| Command(s) to execute

| `description`
| string
| Human-readable description

| `context`
| string
| Execution context to use

| `dir`
| string
| Working directory

| `env`
| map
| Environment variables

| `envfile`
| object
| Environment file configuration

| `variables`
| map
| Template variables

| `condition`
| string
| Condition to check before running

| `timeout`
| duration
| Maximum execution time

| `allow_failure`
| boolean
| Continue pipeline on failure

| `interactive`
| boolean
| Enable interactive mode (TTY)

| `variations`
| array
| Run task multiple times with different env

| `before`
| array
| Commands to run before main command

| `after`
| array
| Commands to run after main command

| `artifacts`
| object
| Output artifact configuration

| `required`
| object
| Required variables/environment

| `reset_context`
| boolean
| Reset context between variations

| `ci_meta`
| map
| CI-specific metadata

|===

== Commands

Commands can be a single string or an array of strings:

[source,yaml]
----
tasks:
  # Single command
  simple:
    command: echo "hello"
    
  # Multiple commands (run sequentially)
  multi:
    command:
      - echo "step 1"
      - echo "step 2"
      - echo "step 3"
      
  # Multi-line command
  script:
    command: |
      echo "Starting..."
      for i in 1 2 3; do
        echo "Iteration $i"
      done
      echo "Done"
----

== Contexts

Run tasks in specific execution environments:

[source,yaml]
----
contexts:
  alpine:
    container:
      name: alpine:latest

tasks:
  in-container:
    context: alpine
    command: cat /etc/os-release
    
  in-default-shell:
    command: echo "runs in cross-platform shell"
----

See xref:contexts.adoc[Contexts] for detailed configuration.

== Environment Variables

[source,yaml]
----
tasks:
  with-env:
    env:
      MY_VAR: my-value
      DEBUG: "true"
    command: echo "$MY_VAR - debug=$DEBUG"
----

== Conditional Execution

Tasks can have conditions that must pass before execution:

[source,yaml]
----
tasks:
  build-if-changes:
    condition: git diff --exit-code --quiet
    command: go build ./...
    
  deploy-if-main:
    condition: test "$(git branch --show-current)" = "main"
    command: ./deploy.sh
----

The condition command must exit with code 0 for the task to run.

== Task Variations

Run a task multiple times with different environment variables:

[source,yaml]
----
tasks:
  build:
    command: |
      GOOS=${GOOS} GOARCH=amd64 go build -o bin/app_${GOOS} ./cmd/main.go
    env:
      GOFLAGS: -ldflags=-s -ldflags=-w
    reset_context: true
    variations:
      - GOOS: linux
      - GOOS: darwin
      - GOOS: windows
----

This runs the build task 3 times, once for each GOOS value.

Use `reset_context: true` to ensure clean context between variations.

== Before and After Hooks

Run commands before and/or after the main command:

[source,yaml]
----
tasks:
  test:
    before:
      - echo "Setting up test environment"
      - mkdir -p /tmp/test-data
    command: go test ./...
    after:
      - echo "Cleaning up"
      - rm -rf /tmp/test-data
----

NOTE: `after` commands run regardless of whether the main command succeeds. This is useful for cleanup.

TIP: `after` commands are NOT stored as artifacts.

== Timeouts

Set a maximum execution time:

[source,yaml]
----
tasks:
  long-running:
    command: ./slow-process.sh
    timeout: 5m
    
  quick-check:
    command: curl -s http://localhost/health
    timeout: 30s
----

== Allow Failure

Allow a task to fail without stopping pipeline execution:

[source,yaml]
----
tasks:
  optional-lint:
    command: golangci-lint run
    allow_failure: true
----

== Required Inputs

Validate required variables and environment before execution:

[source,yaml]
----
tasks:
  deploy:
    command: |
      echo "Deploying {{ .VERSION }} to {{ .ENVIRONMENT }}"
    required:
      vars:
        - VERSION
      env:
        - AWS_ACCESS_KEY_ID
        - AWS_SECRET_ACCESS_KEY
        - ENVIRONMENT
      args:
        - target
----

[cols="1,3",options="header"]
|===
| Field | Description

| `vars`
| Required template variables (checked against variables and environment)

| `env`
| Required environment variables (checked against merged env from all levels)

| `args`
| Required CLI arguments (passed after `--`)

|===

Validation happens at runtime to allow dynamically set variables. All errors are collected before reporting.

== Interactive Mode

Enable TTY for interactive commands:

[source,yaml]
----
tasks:
  shell:
    command: /bin/bash
    interactive: true
----

== Artifacts

Store task outputs for use by subsequent tasks:

[source,yaml]
----
tasks:
  generate-version:
    command: echo "VERSION=1.2.3" > version.env
    artifacts:
      name: version-info
      path: version.env
      type: dotenv

  use-version:
    command: echo "Using version $VERSION"
----

See xref:artifacts.adoc[Artifacts] for more details.

== Working Directory

Set the working directory for a task:

[source,yaml]
----
tasks:
  test-frontend:
    dir: ./frontend
    command: npm test
    
  test-backend:
    dir: "{{ .Root }}/backend"
    command: go test ./...
----

== Storing Task Output

Task output can be captured as artifacts:

[source,yaml]
----
tasks:
  get-git-sha:
    command: git rev-parse HEAD > sha.txt
    artifacts:
      type: file
      path: sha.txt
----

See xref:artifacts.adoc[Artifacts] for dotenv and env artifact types.

== CI Metadata

Add CI-specific metadata for pipeline generation:

[source,yaml]
----
tasks:
  deploy:
    command: ./deploy.sh
    ci_meta:
      runs-on: ubuntu-latest
      environment: production
      needs-approval: true
----

== Practical Examples

=== Complete Build Task

[source,yaml]
----
tasks:
  build:
    description: "Build the Go application"
    dir: "{{ .Root }}"
    env:
      CGO_ENABLED: "0"
    before:
      - mkdir -p bin
    command:
      - go mod download
      - go build -ldflags="-s -w" -o bin/app ./cmd/main.go
    after:
      - ls -la bin/
    timeout: 5m
    artifacts:
      type: file
      path: bin/app
----

=== Test with Coverage

[source,yaml]
----
tasks:
  test:
    description: "Run tests with coverage"
    command: |
      go test -coverprofile=coverage.out ./...
      go tool cover -html=coverage.out -o coverage.html
    artifacts:
      type: file
      path: coverage.html
----

=== Docker Build

[source,yaml]
----
tasks:
  docker-build:
    description: "Build Docker image"
    required:
      env:
        - DOCKER_REGISTRY
    command: |
      docker build -t ${DOCKER_REGISTRY}/myapp:{{ .VERSION }} .
      docker push ${DOCKER_REGISTRY}/myapp:{{ .VERSION }}
    variations:
      - VERSION: latest
      - VERSION: "{{ .GIT_SHA }}"
----
