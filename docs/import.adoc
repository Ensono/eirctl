= Imports
:toc:
:toclevels: 2

Eirctl supports merging configuration files from both local and remote sources. Imports can be simple strings or structured objects with hash verification and file destination.

[source,yaml]
----
# yaml-language-server: $schema=https://raw.githubusercontent.com/Ensono/eirctl/refs/heads/main/schemas/schema_v1.json

import:
  # Simple string format
  - https://raw.githubusercontent.com/Ensono/eirctl/refs/heads/main/shared/infra/tf.yaml
  - git::ssh://private-repo.org/org123/shared-libs//eirctl/security/scaning.yaml?ref=v0.0.1
  - ./local/relative/to/cli/eirctl.yaml
  - /absolute/path/eirctl.yaml

  # Structured format with hash verification
  - src: https://example.com/shared-tasks.yaml
    hash: "sha256:abc123def456..."

  # File import (written to disk at dest instead of merged)
  - src: https://example.com/scripts/deploy.sh
    dest: ./scripts/deploy.sh
----

== Import Entry Properties

[cols="2,1,3",options="header"]
|===
| Property | Required | Description

| `src`
| Yes
| Source path or URL (string format is shorthand for this)

| `hash`
| No
| SHA256 hash for integrity verification (format: `sha256:<hex_digest>`)

| `dest`
| No
| Destination path — if set, file is written to disk instead of being merged as config

|===

== HTTPS

When importing over HTTPS, make sure the link returns the contents directly. For example, on GitHub use a `raw.githubusercontent.com` URL.

NOTE: Ideally the URL should point to a `.yaml` extension file.

[source,yaml]
----
import:
  - https://raw.githubusercontent.com/Ensono/eirctl/refs/heads/main/shared/build/go/eirctl.yaml
----

== Git

When importing over Git, use SSH with private repositories. You can use Git over HTTPS with public ones if the provider does not support raw content URLs like GitHub.

IMPORTANT: The pattern must follow this regex: `^git::(ssh|https?|file)://(.+?)//([^?]+)(?:\?ref=([^&]+))?$`

=== Supported Protocols

[cols="1,3",options="header"]
|===
| Protocol | Example

| SSH
| `git::ssh://git@github.com/org/repo//path/to/file.yaml?ref=v1.0.0`

| HTTPS
| `git::https://github.com/org/repo//path/to/file.yaml?ref=main`

| FILE
| `git::file:///local/repo//path/to/file.yaml`

|===

TIP: An optional `ref` can be set to point to a branch, tag, or commit SHA.

IMPORTANT: The environment variable `GIT_SSH_PASSPHRASE` should be set if your SSH key is encrypted with a passphrase. Current limitation is that this is global - once set, it will be used across all orgs and repos.

=== Examples

Import from a private repository using SSH:
[source,yaml]
----
import:
  - git::ssh://git@github.com/myorg/shared-configs//eirctl/tasks.yaml?ref=v2.0.0
----

Import from a public repository using HTTPS:
[source,yaml]
----
import:
  - git::https://github.com/Ensono/eirctl//shared/security/eirctl.yaml?ref=main
----

== Filesystem

Supports relative or absolute paths.

[source,yaml]
----
import:
  # Relative path (relative to the config file location)
  - ./shared/common-tasks.yaml
  
  # Absolute path
  - /opt/eirctl/shared/contexts.yaml
----

== Import Merging

When multiple imports define the same task, context, or pipeline, the later import will override the earlier one. The main configuration file takes precedence over all imports.

[source,yaml]
----
import:
  - ./base-tasks.yaml      # Loaded first
  - ./override-tasks.yaml  # Can override tasks from base-tasks.yaml

tasks:
  my-task:                 # Takes precedence over any imported my-task
    command: echo "local"
----

== Shared Definitions

Eirctl provides a set of shared definitions you can import directly:

[cols="2,3",options="header"]
|===
| Import Path | Description

| `shared/build/go/eirctl.yaml`
| Go build tasks and contexts

| `shared/security/eirctl.yaml`
| Security scanning tasks (Sonar, vulnerability scanning)

| `shared/terraform/eirctl.yaml`
| Terraform workflow tasks

|===

Example using shared Terraform definitions:

[source,yaml]
----
import:
  - https://raw.githubusercontent.com/Ensono/eirctl/refs/tags/0.9.1/shared/terraform/eirctl.yaml

pipelines:
  deploy:
    - task: eirctl:tf:init
      env:
        TF_DIR: infrastructure
    - task: eirctl:tf:plan
      depends_on: eirctl:tf:init
    - task: eirctl:tf:apply
      depends_on: eirctl:tf:plan
----

== Global Configuration

eirctl loads a global configuration file from `~/.eirctl/config.yaml` if it exists. This allows you to define shared contexts, tasks, or variables that are available across all projects.

The global config is loaded and merged with the project configuration. Project-level definitions take precedence over global ones.
