# Security Policy

## Supported Versions

Security fixes are provided for the latest released version of eirctl. Please upgrade to the latest release before reporting an issue where possible.

## Reporting a Vulnerability

Please report suspected security vulnerabilities privately to [stacks@ensono.com](mailto:stacks@ensono.com). Do not open a public GitHub issue for a potential vulnerability.

Include, where possible:

- a description of the issue and its potential impact;
- steps to reproduce or a proof of concept;
- affected eirctl versions and environments; and
- any suggested mitigation or fix.

We will acknowledge the report, assess its impact, and coordinate remediation and disclosure with the reporter. Please allow us reasonable time to investigate and release a fix before discussing the issue publicly.
